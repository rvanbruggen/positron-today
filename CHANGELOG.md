All notable changes to Positron Today. Newest first.

| **2.15.1** | Fix the wrong Bluesky handle on the Contact page. The EN / NL / FR sections of `site/src/contact.njk` all linked to `https://bsky.app/profile/positrontoday.bsky.social` and displayed `@positrontoday.bsky.social`, but the actual profile lives at the custom domain handle — `@positron.today` / `https://bsky.app/profile/positron.today`. No other files reference the old handle |
| **2.15.0** | Mobile / PWA homepage redesign so phone visitors see several articles above the fold instead of one full card filling the viewport. Under `@media (max-width: 699px)`, the top-featured section in `site/src/index.njk` flips `.home-top-featured` to `flex-direction: row` so the two featured cards render as side-by-side hero tiles (image on top via the existing `.card-img` bleed rule, title below, everything else — summary, emoji, tags, meta row — hidden). Every other card in the main feed picks up a new `card-mobile-compact` class that collapses it into a flex-row: 80×80 square thumbnail (`aspect-ratio: 1/1`, `object-fit: cover`) on the left, 3-line-clamped title on the right. The main feed is also forced back to 1 column on phones (`calc(100% - 1rem)` overrides the 480px+ 2-col Masonry rule) so compact rows are full-width and tap-friendly. Result on iPhone 375×812: 5 articles visible above the fold (2 hero tiles + 3 compact rows), up from 1 full card previously. Mobile also swaps the active click target on every card: two `<a class="card-link">` elements now render per card — `.card-link-desktop` (`target=_blank` → `source_url`) and `.card-link-mobile` (→ `post.url`, the Positron intermediate page) — toggled by `display:none` inside the same mobile media query. Rationale: tapping an external link from a standalone PWA drops the user out of the app shell, and on mobile the inline summary is hidden anyway, so the intermediate page is the natural next step; the Positron page still has a prominent "go to source" button so it's one extra tap, not a dead end. Desktop (≥1100px) and tablet (700–1099px) are completely untouched — full cards with summaries in their respective layouts, click still opens the source in a new tab |

Also clean up the documentation: the 70+ row Version History table moved out of `README.md` into its own `CHANGELOG.md` (this file), cutting README from 524 → 450 lines; the Version Tracking section in README now lists the four canonical files to update on every bump (`admin/lib/version.ts`, `admin/package.json`, `README.md` badge, `CHANGELOG.md` new row) and drops the stale reference to the root `package.json` as a sync target (it hasn't been touched since 2.8.1) |
| **2.14.2** | Require an RSS feed URL on every source added via the Sources admin page, now that one-off URL submission lives on the Preview tab (→ `/api/manual-url` → auto-created "Manual" source). | **2.13.1** | Two polish fixes on top of 2.13.0. Trim the desktop featured rail so it ends roughly where the main feed ends — previously, when there were far more featured articles than regulars the right column ran much longer than the left, leaving a big whitespace hole at the bottom of the left column. | **2.13.0** | Overhaul how the homepage surfaces featured articles so the "wider column" messiness goes away. Out: the old `.card-featured` double-width class that forced Masonry to shuffle 2-column-wide cards into a 1- to 4-column grid and left visible white-space gaps when a wider card couldn't fit the remaining row. In: a region-based architecture using CSS Grid. Desktop (≥1100px) runs a 60/40 split — a 2-column Masonry feed on the left, a dedicated featured rail on the right with top-2 featured above and featured #3+ stacked below. Tablet (700–1099px) shifts the top 2 featured into a 2-column strip above the 2-col feed; the rail is hidden. Mobile (<700px) stacks the top 2 featured and flows the rest of the feed single-column below. Featured cards themselves get a visual treatment (larger headline, image-only, no emoji) by virtue of living in a featured region — no more width-based differentiation, so Masonry never has to place mixed-width cards and no more gaps. Featured #3+ is duplicated into the main feed DOM with `.feed-dup-featured` (hidden on desktop via CSS) so tablet/mobile still surface them, styled as regular cards. Filters (tag + date) now apply to all three regions in lockstep. Empty-rail fallback borrows the most-recent non-featured posts when fewer than 2 featured exist, as a safety net |
